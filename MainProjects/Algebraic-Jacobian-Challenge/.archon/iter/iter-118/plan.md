## The correctness finding

The iter-117 trim refactored `Differentials.lean:74` to the presheaf
form:

```
theorem smooth_iff_locally_free_omega (f : X ⟶ S)
    (hfp : LocallyOfFinitePresentation f) (n : ℕ) :
    IsSmoothOfRelativeDimension n f ↔
      ∀ x, ∃ U, x ∈ U.1 ∧ IsAffineOpen U ∧
        Module.Free R M ∧ Module.rank R M = n
```

The `↔` is **mathematically false** in general. Counterexample:

- Let `k` be a field and consider `f : Spec k → Spec k[t]` induced by
  the surjection `k[t] → k`, `t ↦ 0`.
- `LocallyOfFinitePresentation f`: ✓ (`k = k[t]/(t)` is f.p. over
  `k[t]`).
- For every `x ∈ Spec k` (only one point), and every affine open `U`
  containing `x` (only `U = Spec k`), the relative cotangent module
  `Ω_{k/k[t]} = 0`, which is `Module.Free` of rank 0.
- The RHS of the iff therefore holds for `n = 0`.
- But `f` is a closed immersion of a non-open point; it is not flat,
  hence not smooth. So `SmoothOfRelativeDimension 0 f` is **false**.
- Therefore the iff has its `←` direction **false** for this
  instance of `f` and `n = 0`.

Mathlib's converse-direction lemma
`Algebra.IsStandardSmooth.iff_exists_basis_kaehlerDifferential`
(`Mathlib.RingTheory.Smooth.StandardSmoothOfFree`) makes the
necessary extra hypothesis explicit: `Subsingleton (Algebra.H1Cotangent
R S)` (formal smoothness / vanishing André–Quillen `H¹`). This is
NOT implied by `LocallyOfFinitePresentation` + `Module.Free Ω` of
the right rank.

Forward direction is correct and provable from the verified Mathlib
bridges (see § "Mathlib-name verification").

Per the user directive (iter-116 hint: "no temporarily wrong
statements"), we MUST fix the signature THIS iter. The fix:

- **Lean signature** (`Differentials.lean:74`): refactored to a
  forward-only implication. New shape:

  ```
  theorem smooth_locally_free_omega {n : ℕ} (f : X ⟶ S)
      [SmoothOfRelativeDimension n f] :
      ∀ x, ∃ U, x ∈ U.1 ∧ IsAffineOpen U ∧
        Module.Free R M ∧ Module.rank R M = n
  ```

  (The `LocallyOfFinitePresentation` premise is implied by
  `SmoothOfRelativeDimension n`; we drop it.)

- **Blueprint chapter** (`Differentials.tex`): theorem renamed (or
  the iff is split into a forward direction theorem and a remark
  documenting the converse-direction gap). The forward-direction
  proof sketch is corrected to cite the actually-existing Mathlib
  names (see below). The converse direction is honestly disclosed
  as out-of-scope (Mathlib gap: `Subsingleton (Algebra.H1Cotangent
  A B)` is a real deformation-theoretic hypothesis, not an automatic
  consequence of local freeness + finite presentation).

This is NOT a backwards-compatibility shim: nothing in the project
consumes `smooth_iff_locally_free_omega` (grep confirms: only the
declaration site references it). The presheaf-form was a iter-117
deliverable; we deliver the **correct** version this iter.
