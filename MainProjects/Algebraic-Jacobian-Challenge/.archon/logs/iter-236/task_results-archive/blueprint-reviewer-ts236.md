4. `\theorem` `\label{thm:cm_regularity_global_generation}` — An m-regular sheaf is (globally) generated by its global sections after twisting by m. `\lean{AlgebraicGeometry.IsCMRegular.globallyGenerated}` [expected]. Source: Nitsure §5.

### Proposed chapter: `blueprint/src/chapters/Picard_SemiContinuity.tex`

**Covers**: `AlgebraicJacobian/Picard/SemiContinuity.lean`
**Strategy phase**: A.2.c-engine (Quot/Cartier, RR-free; currently HOLD)
**Why now**: Writing the blueprint now enables clean deferral record-keeping and future dispatcher-parallel dispatch once the engine re-opens.

**Key declarations** (in dependency order):
1. `\definition` `\label{def:semicont_function}` — The function s ↦ dim_κ(s) H^i(X_s, F_s) on the base S is the i-th cohomology dimension function. `\lean{AlgebraicGeometry.cohomDimFunction}` [expected]. Source: Stacks coherent.tex §02KH.
2. `\theorem` `\label{thm:cohom_dim_upper_semicontinuous}` — Under flatness + proper + f qcqs, the function s ↦ dim H^i(X_s, F_s) is upper semicontinuous on S. `\lean{AlgebraicGeometry.isUpperSemiContinuous_cohomDimFunction}` [expected]. Source: Hartshorne III §12 Theorem 12.8; retrieval needed.
3. `\corollary` `\label{cor:euler_char_locally_constant}` — The Euler characteristic s ↦ χ(F_s) is locally constant on S under flatness + proper + qcqs. `\lean{AlgebraicGeometry.isLocallyConstant_eulerChar}` [expected]. Source: Hartshorne III §12 Corollary 12.9.

**`\uses` skeleton**:
- `cor:euler_char_locally_constant` uses `thm:cohom_dim_upper_semicontinuous`, `def:higher_direct_image`, `def:semicont_function`
- `thm:cohom_dim_upper_semicontinuous` uses `def:higher_direct_image`, `def:semicont_function`, `lem:higher_direct_image_quasi_coherent`

**Main theorem proof strategy**: The semicontinuity theorem reduces (by Grauert's theorem) to: given a perfect complex on S, the dimension of its cohomology is upper semicontinuous. The standard Lean route would go through flat base change (`thm:flat_base_change_pushforward`) + fibre cohomology Čech computation on affine opens. The spectral sequence route (as in Hartshorne) requires the Čech-to-sheaf spectral sequence, which may require substantial infrastructure.

**References for writer**:
- `references/stacks-coherent.tex` — Stacks cohomology-of-schemes, specifically the semicontinuity material (tags near 02KH area).
- Hartshorne III §12 — retrieval needed (no local file covering §12 verbatim). Plan agent should dispatch a reference-retriever for Hartshorne §12 before the writer.

**Subphase choices exposed**:
- Full generality (all i, flat + proper) vs. the curve-specific case (i = 0,1, C smooth proper): the curve-specific case is sufficient for the Picard-scheme engine and is substantially simpler. Recommendation: start with curve-specific (simpler, sufficient for engine); generalize only if a consumer needs it.
