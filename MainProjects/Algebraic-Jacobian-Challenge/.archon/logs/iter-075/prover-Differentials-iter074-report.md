### Why the chain is hard to mechanise this iteration

- **No `Derivation.postcomp_comp` lemma in Mathlib.** Step (1) needs to split the
  Derivation `d.postcomp (αv ≫ pushed-β)` into `(d.postcomp αv).postcomp pushed-β`.
  Mathlib's `Derivation.postcomp` is a one-step definition with no chain helper.
  A project-local helper lemma is the pragmatic fix:
  ```
  lemma Derivation.postcomp_comp {d : M.Derivation φ} {f₁ : M ⟶ N} {f₂ : N ⟶ K} :
    d.postcomp (f₁ ≫ f₂) = (d.postcomp f₁).postcomp f₂ := by ...  -- by definitional unfold
  ```
  Once this lands, `simp [Derivation.postcomp_comp, isUniversal'.fac]` should
  reduce the LHS to `d_target.postcomp ((pushforward _).map β.val)`.

- **β's universal property requires unfolding `cotangentExactSeqBeta`.** Step (3)
  needs the fact that `β.val = (isUniversal' φ_fg').desc d1` where
  `d1.d := (derivation' φ_2').d`. Inside the proof of `cotangentExactSeq_structure`,
  `d1` is not in scope (it's local to `cotangentExactSeqBeta`'s tactic body). The
  pragmatic route is to extract a project-local definition for `β.val`'s desc
  Derivation (call it `cotangentExactSeqBeta.d1`) and reference it directly.

- **Build env broken since iter-069** prevents iterative trial-and-error. Each
