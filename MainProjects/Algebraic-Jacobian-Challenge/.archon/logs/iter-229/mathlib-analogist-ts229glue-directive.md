## Proposed declaration / design under review

We need to build, in the project file `AlgebraicJacobian/Picard/TensorObjSubstrate.lean`:

```
AlgebraicGeometry.Scheme.Modules.homOfLocalCompat :
  {X : Scheme} {M N : X.Modules} {ι : Type} (U : ι → Opens X) (hU : ⨆ i, U i = ⊤)
  (f : Π i, M.restrict (U i).ι ⟶ N.restrict (U i).ι)
  (compat : ∀ i j, ... agree on overlaps ...)  →  (M ⟶ N)
```

i.e. **glue a family of compatible local morphisms of sheaves-of-modules (one per open
`Uᵢ` of a cover) into a single global morphism `M ⟶ N`** in `X.Modules =
SheafOfModules X.ringCatSheaf`. This is the "morphism descent" A-bridge of a larger
⊗-inverse construction.

The project's intended route (by inspection, ~120–190 LOC, UNVERIFIED):
1. Form the internal-hom **presheaf of types** `presheafHom (M.val.presheaf)
   (N.val.presheaf)` and note it is a `TopCat.Sheaf` of types via `Presheaf.IsSheaf.hom`
   (needs `N.val` a sheaf).
2. Each local `fᵢ : M.restrict (Uᵢ).ι ⟶ N.restrict (Uᵢ).ι` yields a **section**
   `localSection i : presheafHom(...).obj (op Uᵢ)`. Its component at `(V, h : V ⟶ Uᵢ)` is
   an `eqToHom`-conjugated `fᵢ.mapPresheaf` component using `(Uᵢ).ι ''ᵁ ((Uᵢ).ι ⁻¹ᵁ V) = V`
   for `V ≤ Uᵢ`. **The `naturality` field of `localSection` is the load-bearing coherence
   step** flagged as the only real risk.
3. Glue via `TopCat.Sheaf.existsUnique_gluing` (the `localSection`s agree on overlaps by
   `compat`) to a global section = an ab-sheaf morphism `M.val.presheaf ⟶ N.val.presheaf`.
4. Re-attach 𝒪_X-linearity via the already-built `Scheme.Modules.homMk` (A-bridge step ii)
   to land the morphism in `X.Modules`.
