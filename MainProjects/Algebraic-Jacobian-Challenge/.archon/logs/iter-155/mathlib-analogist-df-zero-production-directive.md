# Mathlib Analogist Directive

## Mode: cross-domain-inspiration

## Slug
df-zero-production

## Structural problem

We must **produce the vanishing of a differential** at the scheme level, as the
keystone input to an already-built rigidity bridge.

Setup: `k̄` algebraically closed, char 0. `C` a smooth proper geometrically
irreducible curve over `k̄` of genus 0 (so `≅ ℙ¹_{k̄}`). `A` a smooth proper
geometrically irreducible group scheme over `k̄` of dimension `g` (an abelian
variety), with identity `η_A`. A morphism `f : C → A` over `Spec k̄` with
`f(p) = η_A` at a rational point `p`. We want to prove `f` is the constant
morphism at `η_A`.

The project has **already built** the "downstream half" of the classical argument:
"if the differential `df = 0` then `f` factors through `Spec k̄` (is constant)"
— realised chart-by-chart via `KaehlerDifferential.mem_range_algebraMap_of_D_eq_zero`
(KDM, closed, axiom-clean) and `df_zero_factors_through_constant_on_chart`.

The **missing input** is `df = 0` itself, i.e. the vanishing of
`df : f^* Ω_{A/k̄} → Ω_{C/k̄}` (equivalently, for every chart generator `b` of an
affine chart `B = Γ(A-chart)`, the Kähler image `D_{Γ(C)}(f^♯ b) = 0`).

Classically `df = 0` because:
  (i)  `Ω_{A/k̄}` is **globally trivial** — generated by translation-invariant
       differentials, `Ω_{A/k̄} ≅ O_A^{⊕g}` — so `f^* Ω_{A/k̄} ≅ O_C^{⊕g}`; and
  (ii) `Hom_{O_C}(O_C^{⊕g}, Ω_{C/k̄}) = H^0(C, Ω_{C/k̄})^{⊕g} = 0` because `C`
       has genus 0 (`Ω_{ℙ¹} ≅ O(−2)`, no global sections).
A map out of a trivial bundle into a sheaf with no global sections is `0`.

We want the **minimal, Mathlib-idiomatic assembly** that establishes `df = 0`
from present in-tree material, ideally avoiding re-building the expensive global
presheaf-of-modules globalisation that was excised (see Failed approaches).

## Failed approaches

- **Global cotangent globalisation of a group scheme (piece (i.b),
  `mulRight_globalises_cotangent`)** — the presheaf-of-modules construction that
  would globalise the cotangent-at-identity to `Ω_A ≅ O_A^{⊕g}` via a functorial
  shear/translation isomorphism was **EXCISED from the Lean tree (iter-145)** as
  too expensive (~800–1500 LOC, presheaf-vs-sheaf bridge cost). We do NOT want to
  rebuild that wholesale if a lighter route exists.
- **The chart-algebra pivot (iter-144)** built only the converse "`df=0` ⟹ constant"
  (piece (ii)), implicitly assuming `df=0` would be available; a fresh blueprint
  review (iter-155) confirmed nothing in the present material produces `df=0`.
  So this is a genuine, ≥1-iter-old structural gap on the critical path, not a
  missing single lemma.

   `HModule` cohomology stack? Is there a Mathlib statement connecting genus 0 (or
   `H^1(C,O_C)=0`) to `H^0(Ω_C)=0` we can lean on, or must it route entirely through
   the project's own cohomology machinery?

   in Mathlib (any domain — module theory, vector bundles, sheaf cohomology,

4. **Sidestep check.** Is there any Mathlib lemma that would let us bypass `df`

   curves on abelian varieties are constant"? State VERIFIED-present or
   ABSENT for each; we expect most to be ABSENT in `b80f227` and want that confirmed
