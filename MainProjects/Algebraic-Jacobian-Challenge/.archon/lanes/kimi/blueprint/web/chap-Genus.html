2 Genus of a smooth proper curve

Throughout this chapter, \(k\) denotes a field and \(C\) a smooth proper geometrically irreducible scheme over \(k\) of relative dimension \(1\) — that is, \(C \to \Spec k\) carries the typeclasses [SmoothOfRelativeDimension 1 C.hom] [IsProper C.hom] [GeometricallyIrreducible C.hom]. We call such a \(C\) a smooth proper curve over \(k\).

2.1 Definition

The genus of a smooth proper curve \(C\) over \(k\) is the natural number

\[ g(C) \; =\; \dim _k H^1(C,\, \mathcal O_C), \]

the \(k\)-dimension of the first sheaf-cohomology group of the structure sheaf of \(C\).

Equivalent reformulations available once Riemann–Roch and Serre duality are formalised:

  • \(g(C) = \dim _k H^0(C,\, \Omega ^1_{C/k})\) (Serre duality).

  • \(g(C) = 1 - \chi (\mathcal O_C)\), where \(\chi \) is the Euler characteristic.

  • For a smooth projective curve over \(\mathbb C\), \(g(C)\) equals the topological genus of \(C(\mathbb C)\).

Each of these requires Mathlib infrastructure that is currently absent (see Section 2.2).

2.2 Mathlib gap

The honest definition \(\dim _k H^1(C, \mathcal O_C)\) requires:

  • Sheaf cohomology of quasi-coherent sheaves on schemes. Mathlib has the abstract right-derived-functor machinery (CategoryTheory.Functor.RightDerived) but no instantiation to \(\Gamma (X, -) \colon \mathrm{QCoh}(X) \to \mathrm{Mod}_{\mathcal O_X(X)}\). Only étale/\(\ell \)-adic cohomology exists (AlgebraicGeometry.Scheme.EllAdicCohomology), which is not what we need.

  • Finite-dimensionality of \(H^i(C, \mathcal F)\) for coherent \(\mathcal F\) on a proper scheme over a field (Serre’s theorem). Absent.

2.2.1 Discovery objective for this iteration

The prover should:

  1. Search Mathlib (lean_local_search, lean_leansearch, lean_loogle) for any existing API that computes \(H^1(C, \mathcal O_C)\) as a \(k\)-vector space, including via Čech cohomology, hypercoverings, or derived functors.

  2. Document the precise API gap in task_results/Genus.md: which type or function would have to exist for the definition to typecheck, and what the closest existing Mathlib declaration is.

  3. Do not introduce a placeholder definition (such as genus := 0 or an opaque Classical.choice); leave the sorry in place rather than weakening the mathematical content.

  4. If — and only if — the cohomology API is found in Mathlib, fill the sorry with the honest definition.

The phase-A roadmap in STRATEGY.md tracks how the missing cohomology API is to be built across subsequent iterations.
