## Decisions identified

### Decision 1: minimal hypotheses for `openImmersion_beckChevalley`
- **Mathematics**: the map is `g'^* p_* F → p'_* gV^* F` for `p` an open immersion (= flat) and `g'`
  arbitrary. This is FLAT BASE CHANGE (Stacks 02KH/02KG). The outer `g'^*` is sheafification of the
  presheaf tensor `(p_*F)(U) ⊗_{O(U)} O'(W)`; that tensor description only matches `p'_* gV^* F`
  sections when the sheaf is QUASI-COHERENT. For a general `O_X`-module the comparison FAILS. ⇒ the
  arbitrary-`F` signature is UNSOUND / not provable; `hF : F.IsQuasicoherent` is required.
- **`[QuasiCompact p]`**: the blueprint hand route (`lem:openimm_pushforward_qcoh_tilde`) presents the
  GLOBAL `p_* p^* F` as `Ñ` over an affine `U`, which needs `p_* p^* F` quasicoherent. Pushforward
  preserves qcoh only under qcqs (Stacks 01XJ; project's only such lemma is `flatBaseChange_pushforward_isIso`,
  gated `[QuasiCompact f][QuasiSeparated f]`; `pushforward_iso_preserves_qcoh` handles ONLY isos).
  Open immersions are always quasi-SEPARATED but NOT quasi-compact (the blueprint proof's claim "an
  open immersion is quasi-compact" is a soundness gap — true only when `V` is retrocompact, e.g. `V`
  affine + `X` quasi-separated). So the HAND route silently needs `[QuasiCompact p]`. The BRICK route
  (Decision 2) does NOT: it pushes forward affine-local tildes, never forming the global `p_* p^* F`,
  and pushforward of a tilde along a `Spec`-map is qcoh by construction. ⇒ with the brick route only
  `hF` is needed.
- **Sibling alignment**: `pullback_isQuasicoherent`, `pullbackRestrict_iso_tilde`,
  `pushPullObj_pushforward_iso_tilde` are ALL `hF`-gated. `openImmersion_beckChevalley` is the lone
  `hF`-free outlier in the tilde family.
- **Gap**: divergent-and-wrong (the arbitrary-`F` statement is false).
- **Verdict**: ALIGN_WITH_MATHLIB — re-sign with `hF : F.IsQuasicoherent`. Do NOT add `[QuasiCompact p]`
  (avoidable via brick route). The lemma is not frozen in `archon-protected.yaml`, so the re-sign is allowed.

### Decision 2: does `[IsAffine V]` collapse the member node to the affine brick?
- **The collapse (YES, for the per-affine member node)**: check `IsIso bareBC` over an affine cover
  `{W_j = Spec B}` of `X'` refining `(g')⁻¹(affine opens of X)`, so `g'(W_j) ⊆ U_j = Spec A`. With
  `V` affine and `X` separated (affine diagonal), `V ∩ U_j` is affine `= Spec A'`, and the restricted
  square over `(W_j, U_j)`
  ```
    W_j ×_{U_j}(V∩U_j) --→ V∩U_j
        |                    | (open immersion = Spec(A→A'))
        v                    v
       W_j  ----Spec ψ---→  U_j
  ```
  is a FULLY-AFFINE pushout square. `openimm_bareBC_app_isIso_affine` over `W_j` is then EXACTLY
  `affinePushforwardPullbackBaseChange` with `f = Spec(A→A')` (the open immersion `V∩U_j↪U_j`) and
  `M = Γ(V∩U_j, p^*F)` (an `A'`-module, presented because `p^*F = F|_V` is qcoh from `hF`). The brick
  ALREADY packages `pullback_spec_tilde_iso + pushforward_spec_tilde_iso + cancelBaseChange`, so it
  REPLACES the three hand leaves `openimm_outer_pullback_affine_tensor` / `openimm_pushforward_sections_basechange`
  / the cancelBaseChange-matching with ONE brick call. Crucial subtlety: pushforward of `p^*F` along
  `p` and restriction-to-`U_j` commute for ARBITRARY sheaves (just `(p_*G)|_{U_j} = (V∩U_j↪U_j)_*(G|_{V∩U_j})`,
  the open-restriction/`pushforward_obj_obj` identity), so the brick never needs `p_*p^*F` qcoh ⇒ no qc.
- **What `[IsAffine V]` does NOT do**: it does NOT make `openImmersion_beckChevalley` an affine-brick
  one-liner. `X'` is still NON-affine, so the brick (affine target) cannot apply to the X'-level square;
  you STILL need the iter-328 keystone `isIso_iff_isIso_restrict` to localize the X'-level `IsIso` to
  the `W_j`, plus the refining affine cover (`openImmersion_refiningAffineCover`, NOT yet built — pull
  the standard affine cover of `X` back along `g'`, refine each preimage by its own affine cover), plus
  the per-`W_j` local-square construction (an analog of `restrictedCartesianAffinePushout` over `W_j`
  with `V∩U_j`, distinct from the existing global S-affine one). In the CONSUMER the refining cover is
  free: the base-changed cover `𝒰'` is already affine with `g'(𝒰'_i)⊆𝒰_i` affine.
- **vs. fbc327 no-affine route**: fbc327 builds the SAME per-`W_j` tensor matching (cancelBaseChange
  for the local square) BY HAND without the brick — over-general and re-derives what the brick packages.
- **Gap**: divergent-with-cost (fbc327 route is the costlier sibling of the brick route).
- **Verdict**: DIVERGE_INTENTIONALLY from fbc327's hand-matching → use the affine-V brick route. It is
  materially cheaper (one sorry-free brick call per member vs. 3 hand leaves), at the cost of `hF` +
  `[IsAffine V]` + X-separatedness threading.

## Recommendation
ALIGN: re-sign `openImmersion_beckChevalley` with `hF : F.IsQuasicoherent` (the arbitrary-`F` form is
unsound — flat base change fails off the qcoh locus) and `[IsAffine V]` (+ X-separatedness). Prove it
via the iter-328 keystone `isIso_iff_isIso_restrict` over a refining affine cover `{W_j}` of `X'`,
discharging each member with `affinePushforwardPullbackBaseChange` on the local fully-affine pushout
square (`f = Spec(A→A')` for `V∩U_j↪U_j`, `M = Γ(V∩U_j, p^*F)`). This REPLACES the effort-broken hand
leaves `openimm_outer_pullback_affine_tensor` / `openimm_pushforward_sections_basechange` /
`openimm_bareBC_app_isIso_affine`-via-cancelBaseChange with one sorry-free brick call per member, and
AVOIDS `[QuasiCompact p]` (the brick never forms the global `p_*p^*F`). The remaining net-new work is
`openImmersion_refiningAffineCover` (pull std affine cover of X back along g', refine — Mathlib
ingredients only; in the consumer it is the existing `𝒰'`) and the per-`W_j` local affine pushout
square. fbc327's no-affine route is over-general and should be retired in favour of this. Related:
[[fbc327]], [[fbc-keystone-landed-328]], [[fbc-affine-bridge-325]], [[fbc-isiso-barebc-327]].
