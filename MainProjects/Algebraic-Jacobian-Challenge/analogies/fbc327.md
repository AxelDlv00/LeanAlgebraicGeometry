## Decisions identified

### Decision: Q1 — does Mathlib describe `SheafOfModules.pullback` sections affine-locally?
- **Mathlib idiom**: `SheafOfModules.pullback` (`Mathlib.Algebra.Category.ModuleCat.Sheaf.PullbackContinuous`)
  is the **abstract left adjoint**, realized as `SheafOfModules.pullbackIso`:
  `pullback φ ≅ forget ⋙ PresheafOfModules.pullback φ.hom ⋙ sheafification` — i.e. *sheafification of the
  presheaf-tensor*. Concrete sections exist ONLY for FREE modules (`pullbackObjFreeIso`,
  `Mathlib...PullbackFree`) and the unit (`pullbackObjUnitToUnit`). The single concrete *non-free* handle is
  `AlgebraicGeometry.Scheme.Modules.restrictFunctorIsoPullback` (`Mathlib.AlgebraicGeometry.Modules.Sheaf`):
  for an **open immersion** `f`, `pullback f ≅ restrictFunctor f` (= restriction, concrete sections).
  There is NO `pullback_app` / affine-open tensor-sections lemma for arbitrary `g'`. Mathlib has NO
  sheaf-level Beck–Chevalley; only the ALGEBRAIC `Algebra.IsPushout.cancelBaseChange` /
  `Module.Flat.isBaseChange`.
- **Project's path**: project already owns the literal-Spec instance `pullback_spec_tilde_iso`.
- **Gap**: divergent-and-wrong-to-expect — the general formula does not exist.
- **Verdict**: NEEDS_MATHLIB_GAP_FILL (Stacks 01HQ/0BJ8 — the pullback-section tensor formula is the gap,
  shared with `canonicalBaseChangeMap_isIso`).

### Decision: Q2 — abstract-affine pullback-tilde bridge / widen with base datum?
- **Idiom**: the pushforward bridge `pushPullObj_pushforward_iso_tilde_affine` transports the literal-Spec
  formula along `S.isoSpec` because its morphism `f : X ⟶ S` has an **affine target `S`**.
- **Project's path (proposed)**: same transport for `pullback_spec_tilde_iso`.
- **Gap**: divergent-and-wrong for THIS residual. `openImmersion_bareBC` / `openImmersion_beckChevalley`
  is **X-level**: the outer pullback is `g' : X' ⟶ X` with X, X' **non-affine**. A tilde-pullback bridge
  needs `g'` to BE a `Spec.map` (affine source AND target) — it is not. Threading the consumer's base datum
  (`f,g,f',g',h,[IsAffine S],[IsAffine S']`) makes `V = U_σ` and `V' = X'×_X V` affine but leaves X, X'
  non-affine, so `IsIso bareBC` is NOT an affine-local statement. ⇒ widening `openImmersion_beckChevalley`
  with the base datum does NOT create an affine route.
- **Cost of pursuing**: building the pullback-tilde bridge is a fine reusable lemma BUT does not close
  `IsIso bareBC`; spending iters on it as a bypass is wasted.
- **Verdict**: PROCEED-to-build-bridge-but-DOES-NOT-CLOSE-residual (DEAD as a bypass).

### Decision: Q3 — re-route heart RHS through the LHS affine bridge?
- **Idiom**: LHS bridge handles a **pushforward-only** term `f_*(p_* p^* F)` by collapsing `f_* p_* = (p≫f)_*`.
- **Project's path (proposed)**: do the same on the RHS `f'_*(g'^*(p_* p^* F))`.
- **Gap**: divergent-and-wrong. The RHS has `g'^*` **sandwiched** between `f'_*` and `p_*`; reducing it to
  pushforward-only form *is* the open-immersion Beck–Chevalley. Concretely: the affine brick
  `affinePushforwardPullbackBaseChange` over the V-restricted affine pushout square proves
  `g^*((f|_V)_* M̃) ≅ (f'|_{V'})_*(gV^* M̃)` (heart-LHS ≅ brick-RHS); converting brick-RHS
  `(f'|_{V'})_*(gV^* M̃) = f'_*((V'.ι)_*(gV^* M))` to heart-RHS `f'_*(g'^*((V.ι)_* M))` reduces, under `f'_*`,
  to exactly `(V'.ι)_*(gV^* M) ≅ g'^*((V.ι)_* M)` = `openImmersion_beckChevalley`. So (3) RE-DERIVES the
  same obligation rather than bypassing it.
- **Verdict**: DEAD as a bypass.

## Why `IsIso bareBC` is the irreducible (true) frontier
- It is TRUE **unconditionally** (the missing `[Flat g']` is correct): an open immersion `p` is FLAT ⟹ the
  cartesian square `IsPullback gV p' p g'` is **tor-independent** ⟹ base change holds for ANY `g'`. So the
  frozen signature is sound; the content is real (tor-independent base change, Stacks 02KG).
- It is genuinely absent from Mathlib at the sheaf level.

## Base-free affine-open route (the recommendation)
There IS a base-free affine-open route — it does NOT widen the signature; it LOCALIZES X':
1. All of `p`, `p'`, `gV` are open immersions (`gV` by `AlgebraicGeometry.isOpenImmersion_stableUnderBaseChange`),
   so `pullback gV ≅ restrictFunctor gV`, `pullback p' ≅ restrictFunctor p'` (`restrictFunctorIsoPullback`):
   the ONLY non-restriction functor in `bareBC` is the outer `g'^*`.
2. Check `IsIso bareBC` sectionwise (`Scheme.Modules.Hom.isIso_iff_isIso_app`) over an **affine open cover
   {Wⱼ} of X' refining `g'⁻¹(affine opens of X)`**: over each affine `Wⱼ` with `g'(Wⱼ) ⊆ affine U ⊆ X`,
   `p_* F = (V.ι)_* F` is quasicoherent over `U`, hence `≅ tilde(N_U)`; then `(g'|_{Wⱼ})^*(tilde N_U) ≅
   tilde(Γ(Wⱼ) ⊗_{Γ(U)} N_U)` by the OWNED `pullback_spec_tilde_iso` (now `g'|_{Wⱼ}` IS a `Spec.map`), and
   the open-immersion side becomes a restriction; matching the two tensors is the algebraic
   `Algebra.IsPushout.cancelBaseChange` (as in the affine brick) for the LOCAL square. This localizes the
   X-level statement to the affine-tilde dictionaries the project already owns.

## Recommendation
ALIGN with the Mathlib reality: `IsIso bareBC` is a NEEDS_MATHLIB_GAP_FILL, NOT closable by an affine
pullback-tilde bridge (Q2) or an RHS reroute (Q3) — both re-derive it. STOP pursuing (2)/(3) as bypasses.
The lower-risk concrete path is the **base-free cover-refinement sectionwise route** above: it needs no
signature widening and reuses `restrictFunctorIsoPullback` + `pullback_spec_tilde_iso` +
`cancelBaseChange`. If that is judged too heavy, the strategic alternative is to restate the 02KH
separated route so the nerve degrees feed the affine brick `affinePushforwardPullbackBaseChange` directly
without ever forming the X-level `openImmersion_beckChevalley` — but note (3) shows the inner BC reappears,
so this is a genuine redesign, not a quick win. Related: [[fbc-pushpull-tilde-317]],
[[fbc-affine-bridge-325]], [[fbc-openimm-bc-stage1-326]].
