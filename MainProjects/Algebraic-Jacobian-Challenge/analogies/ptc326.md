## Question

`PresheafOfModules (Sheaf.val X.ringCatSheaf)` (the `Scheme.PresheafOfModules X` carrier),

`BIG_L = BIG_R` extraction must be done differently?

## THE decisive Mathlib fact (cited)
`PresheafOfModules.monoidalCategoryStruct` (and `.monoidalCategory`),
`Mathlib.Algebra.Category.ModuleCat.Presheaf.Monoidal`:
```
{C} → [Category C] → {R : Cᵒᵖ ⥤ CommRingCat} →
  MonoidalCategoryStruct (PresheafOfModules (R.comp (forget₂ CommRingCat RingCat)))
```
The instance is **keyed on the carrier having the syntactic shape `R ⋙ forget₂ CommRingCat RingCat`**
(`R.comp (forget₂ …)`), with `R : Cᵒᵖ ⥤ CommRingCat`. There is **NO** Mathlib instance keyed on
`Sheaf.val`/`Sheaf.obj`/`FullSubcategory.obj`. The absence is deliberate: Mathlib expects every
monoidal `PresheafOfModules` carrier to be carried in the `⋙ forget₂` form.

Consequence for TC search of `MonoidalCategoryStruct (PresheafOfModules ?carrier)`:
- `?carrier = X.presheaf ⋙ forget₂ CommRingCat RingCat` → discr-tree key `Functor.comp ?R (forget₂ …)`
  matches; unifies `R := X.presheaf`; instance fires. ✓
- `?carrier = Sheaf.val X.ringCatSheaf` (= `X.ringCatSheaf.obj`, the deprecated `.obj` of a
  `(sheafCompose _ _).obj X.sheaf`) → head is `FullSubcategory.obj`, NOT `Functor.comp`; the
  higher-order unification `?R.comp (forget₂ …) =?= Sheaf.val X.ringCatSheaf` is never even tried at
  `instances` transparency. ✗  ← exactly why the `letI inferInstanceAs` bridge is needed.

## Decisions identified

### Decision 1: global `Sheaf.val`-keyed `MonoidalCategoryStruct` instance vs. in-context `letI`
- **Mathlib idiom**: provide the monoidal structure on the **canonical `R ⋙ forget₂` carrier only**
  (`PresheafOfModules.monoidalCategoryStruct`). Never re-key it on an accessor/deprecated wrapper.
  Mathlib has NO precedent for an `inferInstanceAs`-rekeyed monoidal instance on a `def`/accessor-
  wrapped carrier; it instead carries the `⋙ forget₂` spelling everywhere (see `Monoidal.tensorObj_obj`
  etc., all stated at `R.comp (forget₂ …)`).
- **Soundness (kernel) diamond?**: NO. A global
  `instance : MonoidalCategoryStruct (PresheafOfModules (Sheaf.val X.ringCatSheaf)) := inferInstanceAs
  (… (X.presheaf ⋙ forget₂ …))` reduces to the SAME `monoidalCategoryStruct` data (carriers are DEFEQ:
  `Sheaf.val X.ringCatSheaf = X.ringCatSheaf.obj = X.presheaf ⋙ forget₂ …`). For any given **syntactic**
  carrier only one instance's discr-tree key matches (the `Sheaf.val`-keyed one vs. Mathlib's
  `comp`-keyed one are never both candidates for the same syntactic goal), so the kernel never sees two
  non-defeq instances. No unsoundness.
- **The real hazard (TC/metavar diamond)**: YES, and it is the **iter-257 wall itself**. When the
  carrier is a **metavariable** to be inferred from `Fp_h.map`'s expected type (precisely the
  standalone-`⊗ₘ` situation), TC now has TWO matching candidates that produce DEFEQ-but-syntactically-
  DISTINCT `toMonoidalCategoryStruct` projections. The `instances`-transparency unifier may commit
  `⊗ₘ`'s carrier-metavar to the `Sheaf.val` representative while `Fp_h.map` expects the `⋙ forget₂`
  representative → the two `tensorObj`/`toMonoidalCategoryStruct` don't match syntactically → stuck
  metavar (exactly the failure the comment at `:1264-1265` already records when pinning
  `(C := … ⋙ forget₂ …)`). The global instance does NOT dissolve this; it adds a second wrong-shaped
  candidate and can make it more likely.
- **Gap**: divergent-with-cost (parallel API: a second TC path to the structure Mathlib already
  provides, keyed on a DEPRECATED accessor head `Sheaf.val`; re-arms iter-257).
- **Verdict**: **AVOID** the global `Sheaf.val`-keyed instance. **ALIGN_WITH_MATHLIB**: carry the
  canonical `⋙ forget₂` spelling so Mathlib's `monoidalCategoryStruct` fires directly and uniquely.

### Decision 2: explicit instance argument `[inst : MonoidalCategoryStruct (PresheafOfModules …)]`
- **Mathlib idiom**: Mathlib states carrier-ambiguous monoidal lemmas over an **abstract variable
  `C` with `[MonoidalCategory C]`** — the carrier is a variable, never a fixed accessor term. It does
  NOT pass a `MonoidalCategoryStruct` as an *explicit* argument to dodge synthesis on a concrete
  carrier. The `[explicit-instance-arg]` idiom is not used here.
- **Would it dodge the `Fp_h.map` mismatch?**: NO. Passing `instMS` lets the bare *statement*
  elaborate, but `Fp_h.map`'s expected `tensorObj` is resolved by `Fp_h`'s OWN TC search (Mathlib's
  instance at `Fp_h`'s carrier), independent of the supplied `instMS`. Unless the carriers are
  syntactically unified, the mismatch persists. **AVOID.**
- **Verdict**: **AVOID** (not a Mathlib idiom; does not fix the mismatch).

### Decision 3: the actual aligned unblock — re-expose the canonical carrier (no new instance)
`BIG_L = BIG_R` is, mathematically, just `congrArg₂ (·⊗ₘ·) (cmp_leg h f M) (cmp_leg h f N)`
(tensorHom bifunctoriality applied to the two closed legs). The ONLY blocker is making `⊗ₘ` resolve to
Mathlib's `monoidalCategoryStruct`. Two aligned shapes, both using the project's OWN existing idiom
(`TensorObjSubstrate.lean:366`) — NOT a new global instance:

- **3a (in-place, cheapest — RECOMMENDED).** Keep the residual in `pullbackTensorMap_restrict`.
  Immediately before the `refine congr_arg _ ?_` at `:1270`, install the project's bridge:
  ```
  letI instMS : MonoidalCategoryStruct
      (_root_.PresheafOfModules (Z.presheaf ⋙ forget₂ CommRingCat RingCat)) :=
    inferInstanceAs (MonoidalCategoryStruct
      (_root_.PresheafOfModules (Z.presheaf ⋙ forget₂ CommRingCat RingCat)))
  ```
  Then `refine congr_arg _ ?_`, and **`change`/`show`** the residual `BIG_L = BIG_R` to the goal whose
  `⊗ₘ` is spelled at `Z.presheaf ⋙ forget₂ CommRingCat RingCat` (DEFEQ: both `⊗ₘ` instances reduce to
  `monoidalCategoryStruct` over defeq carriers, so `change` succeeds — and per `ptc-carrier-reconcile-325`
  default transparency does NOT whnf-explode here, so do the `change` at default, dropping any
  `respectTransparency true` override). The existing goal's `⊗ₘ` was baked in by Steps A–D under the
  `Sheaf.val` relaxed-transparency instance; the `change` re-keys it to Mathlib's, after which
  `exact congrArg₂ (· ⊗ₘ ·) (cmp_leg h f M) (cmp_leg h f N)` (or `tensorHom_comp_tensorHom`/
  `MonoidalCategory.tensor_comp` + the two legs) fires.

- **3b (extraction, if 3a's `change` is hard to spell).** Extract a standalone lemma whose **statement
  carrier is the canonical `⋙ forget₂` form** (so Mathlib's `monoidalCategoryStruct` fires at
  STATEMENT elaboration — no instance in local context needed):
  ```
  private lemma big_LR {X Y Z : Scheme.{u}} (h : Z ⟶ Y) (f : Y ⟶ X) (M N : X.Modules) :
      (… cmp_leg-LHS M …) ⊗ₘ (… cmp_leg-LHS N …)
        = (… cmp_leg-RHS M …) ⊗ₘ (… cmp_leg-RHS N …) :=
    congrArg₂ (· ⊗ₘ ·) (cmp_leg h f M) (cmp_leg h f N)
  ```
  with every `PresheafOfModules (…)` written `(Z.presheaf ⋙ forget₂ CommRingCat RingCat)`, NOT
  `Sheaf.val Z.ringCatSheaf`. The `cmp_leg` legs (typed at `Z.ringCatSheaf.val`) plug in up to defeq.
  Then close the residual with `exact congr_arg _ (big_LR h f M N)` (or a defeq `change` first). This
  is the cleanest if you want the merges to run at fresh transparency, but it costs transcribing the
  two giant leg expressions; prefer 3a.

## Recommendation
**AVOID** the global `MonoidalCategoryStruct (PresheafOfModules (Sheaf.val ·.ringCatSheaf))` instance
and **AVOID** the explicit-instance-arg lemma shape — both are parallel-API hazards that re-arm the
iter-257 stuck-metavar wall (the global instance re-keys Mathlib's structure on the DEPRECATED
`Sheaf.val` head; the explicit arg doesn't touch `Fp_h.map`'s independent synthesis). The
Mathlib-aligned fix is to carry the **canonical `Z.presheaf ⋙ forget₂ CommRingCat RingCat` carrier**
wherever `⊗ₘ` is written, so the unique Mathlib `monoidalCategoryStruct` fires. Concretely: route 3a —
in-place `letI inferInstanceAs` (the project's own idiom, `TensorObjSubstrate.lean:366`) + a defeq
`change`/`show` to re-expose `BIG_L = BIG_R` at the `⋙ forget₂` carrier, then
`congrArg₂ (·⊗ₘ·) (cmp_leg h f M) (cmp_leg h f N)`. Fallback 3b: extract with the statement carrier
spelled `⋙ forget₂`. **Cost of NOT aligning** (shipping the global instance): a second, deprecated-
accessor-keyed TC path to a structure Mathlib already supplies → metavar-ambiguous `⊗ₘ` synthesis
(iter-257) re-triggers on every future standalone monoidal lemma over the Scheme-`Modules` carrier,
and the instance must eventually be unwound to consume Mathlib's unified API.
