## Answers to the four directive questions

**Q1 — `IsQuasicoherent`/`QuasicoherentData` → global `Presentation`?** No. The whole
`QuasicoherentData` API runs the *other* direction or shuffles local data:
`Presentation.quasicoherentData` (Pres→QCData), `QuasicoherentData.bind`/`.shrink`/
`.localGeneratorsData`, `IsQuasicoherent.of_coversTop`
(`Mathlib/Algebra/Category/ModuleCat/Sheaf/Quasicoherent.lean:201–382`). The global
`GeneratingSections` (`.../Sheaf/Generators.lean:45`) has **no** constructor from the local
`LocalGeneratorsData`. No "globally generated on affine", no "sections generate a QC sheaf".
Confirmed by `grep`: `fromTildeΓ`/`isIso_fromTildeΓ` occur **only** in
`Mathlib/AlgebraicGeometry/Modules/Tilde.lean`, and its own comment (line 310) says the
QCoh = essImage identification "will later be shown".

**Q2 — canonical local→global Presentation gluing idiom?** None in Mathlib. Mathematically
Route A would need (a) QC-on-affine ⟹ globally generated by sections, (b) kernel of `free ⟶ M`
also QC + globally generated, (c) cokernel-of-frees. Step (a) needs the same
section-localization/affine input as the shorter routes *plus more*, so Route A is **strictly
harder**. Recommend against it.

**Q3 — shorter route avoiding `Presentation`?** Yes — skip `isIso_fromTildeΓ_of_presentation`
entirely and prove `IsIso M.fromTildeΓ` by stalk-iso. Named glue, all present:
- `CategoryTheory.Functor.FullyFaithful.reflectsIsomorphisms`
  (`.../Functor/ReflectsIso/Basic.lean:53`) on the project's `SpecModulesToSheafFullyFaithful`:
  reduce to `IsIso (modulesSpecToSheaf.map M.fromTildeΓ)`.
- `TopCat.Sheaf.isIso_of_stalkFunctor_map_iso` (`Topology/Sheaves/Stalks.lean:636`,
  iff at :652): sheaf morphism iso ⟸ iso on every stalk.
- `tilde.toStalk` localization instance (`Tilde.lean:131`): `(tilde N).stalk p ≅ N_p`.
- `IsLocalizedModule.linearEquiv` (`Algebra/Module/LocalizedModule/Basic.lean:923`): identify
  the two localizations to get the component/stalk iso.
- `Scheme.Modules.toOpen_fromTildeΓ_app` (`Tilde.lean:230`): commuting triangle on basic opens.
BUT the shorter route does **not** remove the irreducible content (next paragraph); it only
removes the need to *build a Presentation*.

**Q4 — verdict.** **Genuine multi-step descent.** Every route funnels through one
Mathlib-absent fact (Stacks tag 01HA):
> for QC `M` on `Spec R` and `f : R`, `Γ(M,⊤) → Γ(M,D(f))` is `IsLocalizedModule (powers f)`
> (equivalently `M.stalk p = (Γ(M,⊤))_p`),
proven from `QuasicoherentData` by: refine cover to basic opens (`isBasis_basic_opens`,
finite by `CompactSpace (PrimeSpectrum R)`, `Spectrum/Prime/Topology.lean:291`) → on each
`D(g_a)=Spec R_{g_a}` local presentation + `isIso_fromTildeΓ_of_presentation` ⟹ `M ≅ tilde N_a`
(`isLocalization_basicOpen`, `AffineScheme.lean:650`) → sheaf-equalizer over the finite cover
+ exactness of localization (`IsLocalization.flat`, `Flat/Localization.lean:36`).

## Informational

- **Recommended decomposition for the build lane (Route C, project-aligned):**
  - **G1-core** *(the gap)*: `QC M ⟹ IsLocalizedModule (powers f) (Γ(M,⊤)→Γ(M,D(f)))`. Build
    from `QuasicoherentData` via cover-refine → local-tilde → flat-equalizer. This is *also* the
    missing keystone `lem:qcoh_section_localization_basicOpen`, reused downstream (gap2) — so it
    pays for itself twice.
  - **G1-assemble** *(cheap, pure Mathlib glue)*: from G1-core, each basic-open component
    `N_f → Γ(M,D(f))` of `modulesSpecToSheaf.map M.fromTildeΓ` is iso via
    `IsLocalizedModule.linearEquiv` + `isLocalizedModule_tilde_restrict` (project, already built)
    + `toOpen_fromTildeΓ_app`; basis ⟹ stalk-iso ⟹ `isIso_of_stalkFunctor_map_iso` ⟹
    `FullyFaithful.reflectsIsomorphisms` ⟹ `IsIso M.fromTildeΓ`.

  currently `\uses` `lem:qcoh_affine_isIso_fromTildeΓ` (gap1), and the project's
  `isLocalizedModule_restrict_of_isIso_fromTildeΓ` (`QuotScheme.lean:510`) derives
  section-localization *from* `[IsIso fromTildeΓ]` — backwards. Honest order:
  **G1-core (section-localization from QC data) → gap1 → general-scheme `isLocalizedModule_basicOpen`
